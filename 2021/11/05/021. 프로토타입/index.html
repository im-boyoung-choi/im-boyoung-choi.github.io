프로토타입

개발을 하다보면 기존에 만들어져 있는 객체(또는 기능)와 유사한 객체를 만들어야 하는 경우가 있다. 이럴때 프로토타입 상속(prototypal inheritance)을 이용하면 기존의 객체를 복사하거나 다시 구현하지 않고 확장할 수 있다.

모든 객체는 [[Prototype]]이라는 숨김 프로퍼티를 갖는다. 이 숨김 프로퍼티 값은 null이거나 다른 객체에 대한 참조인데, 다른 객체를 참조하는 경우 참조 대상을 프로토타입(prototype)이라고 한다.

객체에서 프로퍼티를 읽을 때 객체 내에 해당 프로퍼티가 없으면 자동으로 프로토타입에서 프로퍼티를 찾는다. 프로그래밍에서는 이런 동작 방식을 프로토타입 상속(prototypal inheritance)이라고 한다.

JavaScript에서의 상속
JavaScript에서 사용하는 상속은 클래스 기반의 객체 지향 언어에서 사용하는 상속의 개념과 다르다. JavaScript에서는 현재 존재하고 있는 객체를 프로토타입으로 사용하여, 해당 객체를 복제해서 재사용하는 것을 상속이라고 한다.
모든 JavaScript 객체는 Object 객체와 Object.prototype 객체의 프로퍼티를 모두 상속받으며, Object.prototype 객체의 프로퍼티를 이용하면 현재 존재하는 prototype에 새로운 프로퍼티나 메소드를 손쉽게 추가할 수 있다.

[[Prototype]] 프로퍼티는 내부 프로퍼티이자 숨김 프로퍼티지만 __proto__ 문법을 통해 개발자가 값을 설정할 수 있다. 아래 예제를 살펴보자.

rabbit 객체에서 프로퍼티를 읽고 싶은데 해당 프로퍼티가 없다면 프로토타입에서 프로퍼티를 찾는다. 위 예제에서는 rabbit.__proto__ = animal를 통해 rabbit 객체의 프로토타입이 animal 객체가 되도록 설정했다. 프로토타입을 설정하면 rabbit 객체를 호출할 때도 animal 객체의 프로퍼티를 사용할 수 있게 된다. 이렇게 프로토타입에서 상송받은 프로퍼티를 상속 프로퍼티(inherited property)라고 한다.

__proto__
__proto__는 [[Prototype]]과는 다른 존재다. __proto__는 [[Prototype]]의 getter이자 setter이다. 하위 호환성 문제로 __proto__ 대신 Object.getPrototypeOf나 Objet.setPrototypeOf를 사용해서 프로토타입을 획득(get)하거나 설정(set)하기도 한다.

__proto__는 브라우저 환경에서만 지원하도록 자바스크립트 명세서에 규정되어 있는데, 실제로는 서버 사이드를 포함한 모든 호스트 환경에서 지원한다.

1. 프로토타입 체인

프로토타입도 또 다시 상위 프로토타입 객체로부터 프로퍼티와 메소드를 상속 받을 수 있고, 그 상위 프로토타입 객체도 마찬가지다. 이와 같이 프로토타입이 상속되는 가상의 연결고리를 프로토타입 체인(prototype chain)이라 한다. 프로토타입 체이닝에는 다음과 같은 제약사항이 있다.

  1. 순환 참조(circular reference)는 허용하지 않는다. __proto__를 이용해 닫힌 형태로 다른 객체를 참조하면 에러가 발생한다.
  2. __proto__ 값은 객체나 null만 가능하다. 다른 자료형은 무시된다.

2. prototype 프로퍼티와 constructor 프로퍼티

함수 객체의 prototype 프로퍼티는 위에서 설명했던 내부 프로퍼티인 [[Prototype]]과는 약간의 차이가 있다.

  • [[Prototype]] : 자신을 만들어낸 객체의 원형과 연결된 속성
  • prototype 프로퍼티 : 자신을 원형으로 만들어진 새로운 객체들과 연결된 속성
  • constructor 프로퍼티 : 생성자로, 자신을 만들어낸 객체와 연결된 속성

prototype 프로퍼티는 함수가 생성될 때 만들어져 constructor 프로퍼티 하나만 있는 객체를 가리키며, prototype 프로퍼티가 가리키는 프로토타입 객체의 유일한 프로퍼티인 constructor 프로퍼티는 자신과 연결된 함수를 가리킨다.

즉, 함수를 생성할 때 함수 자신과 함께 연결된 프로토타입 객체가 동시에 생성되는 데 각각 prototype 프로퍼티와 constructor 프로퍼티를 가지며 서로를 참조하는 것이다. 아래 코드를 통해 살펴보자.
