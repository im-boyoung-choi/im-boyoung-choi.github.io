연산자

JavaScript에서는 여러 종류의 연산을 위한 다양한 연산자(operator)를 제공한다.

우리가 흔히 알고 있는 산술 연산자(arithmetic operator)부터 증감 연산자(increment and decrement operator), 대입 연산자(assignment operator) 등 다양한 연산자들에 대해 이번 시간에 살펴보자.

1. 산술 연산자

산술 연산자(arithmetic operator)는 사칙연산을 다루는 가장 기본적인 연산자로, +, -, *, /, %가 있다. CodeSandbox에 아래 코드를 입력해 보자.

단순 계산은 물론 혼합 계산도 거뜬하다. 혼합 계산을 할 때는 기본 사칙연산의 우선 순위를 따른다.

% 연산자는 / 연산자와 동일하게 계산한 뒤 나머지를 반환하는 연산자이다. 소수점 자리를 반환하는 것이 아니라 나머지를 반환한다는 것을 잊지말자.

4. 비교 연산자

비교 연산자(comparison operator)는 두 값을 비교할 때 사용하는 연산자이다.

두 값이 일치하는지 확인

=== 연산자는 두 값이 일치하는지 확인한다. 일치하면 true, 아니면 false가 나온다.

비슷한 == 연산자도 확인해보자.

== 연산자도 같은 결과 값이 나왔다. 그러면 두 연산자의 차이는 무엇일까. 바로 == 연산자는 타입 검사는 하지 않는다는 것이다. 타입 검사를 하지 않으면 숫자 타입의 1과 String 타입의 1, 0과 false, null과 undefined 값을 같은 값으로 간주한다.

그래서 두 값이 일치하는지를 확인하고 싶다면 == 연산자보다는 === 연산자 사용을 권장한다.

두 값이 일치하지 않는지 확인

두 값이 일치하지 않는지 확인하려면 !== 연산자를 사용하면 된다.

비슷하게 != 연산자도 있지만 == 연산자와 동일하게 타입 검사를 하지 않는다.

마찬가지로 두 값이 일치하지 않는지 확인하려면 != 연산자보다는 !== 연산자 사용을 권장한다.

5. 논리 연산자

논리 연산자(logical operator)는 blooean 타입(true/false)을 위한 연산자다. 논리 연산자는 if 문을 배울 때 유용하게 사용되니 꼭 기억해두자.

NOT 연산자

NOT 연산자(!)는 true는 false로, false는 true로 바꿔주는 연산자이다.

AND 연산자

AND 연산자(&&)는 양쪽의 값이 모두 true 일 때만 결과물이 true가 나온다.

OR 연산자

OR 연산자(||)는 양쪽의 값이 하나라도 true라도 결과물이 true가 나온다. 즉, 두 값이 다 false여야 false가 나온다.

6. 비트 연산자

비트 연산자(bitwise operator)는 논리 연산자와 비슷하지만 비트(bit) 단위로 논리 연산을 수행한다는 차이점이 있다. 비트 단위로 전체 비트를 왼쪽이나 오른쪽으로 이동시킬 때도 사용한다.

비트 AND 연산자

비트 AND 연산자(&)는 대응되는 두 비트가 모두 1일 때만 1을 반환하며, 다른 경우는 모두 0을 반환한다.

비트 OR 연산자

비트 OR 연산자(|)는 대응되는 두 비트 중 하나라도 1이면 1을 반환하며, 두 비트가 모두 0일 때만 0을 반환한다.

비트 XOR 연산자

비트 XOR 연산자(^)는 대응되는 두 비트가 서로 다르면 1을 반환하고, 서로 같으면 0을 반환한다.

비트 NOT 연산자

비트 NOT 연산자(~)는 해당 비트가 1이면 0을 반환하고, 0이면 1을 반환한다.

시프트 연산자

시프트 연산자(<<, >>)는 이진법을 기준으로 왼쪽 오른쪽으로 비트를 움직여 반환한다. 아래 예시를 살펴보자.

숫자 15는 이진법으로 1111이다. x << 1은 이진법 1111에서 왼쪽으로 1번 시프트하라는 의미이기 때문에 11110이 된다. 따라서 11110의 십진수인 30이 반환된다.

숫자 8은 이진법으로 1000이다. x >> 1은 이진법 1000에서 오른쪽으로 1번 시프트 하라는 의미이기 때무에 100이 된다. 따라서 100의 십진수인 4가 반환된다.

7. 기타 연산자

문자열 결합 연산자

+ 연산자는 문자열 + 문자열, 문자열 + 숫자 일때는 문자열 결합 연산자로 사용된다.

삼항 연산자

삼항 연산자(ternary operator)는 유일하게 피연산자가 3개 존재하는 조건 연산자이다. 표현식 ? 반환값1 : 반환값2의 형태를 가지며 물음표 앞의 표현식 결과 값이 참이면 반환값1을, 거짓이면 반환값2를 반환한다. if-else 문 대신 사용해 코드를 간결하게 만들어 주기도 한다.

쉼표 연산자

쉼표 연산자는 for 문에서 사용하면 루프마다 여러 변수를 동시에 갱신할 수 있는 연산자다. 예제를 통해 살펴보자.

delete 연산자

delete 연산자는 피연산자인 객체, 객체의 프로퍼티(property), 배열의 요소(element) 등을 삭제할 때 사용하는 연산자이다.

피연산자가 성공적으로 삭제됐을 때 true를 반환, 삭제하지 못했을 때는 false를 반환한다.

배열의 요소를 삭제하는 것이기 때문에 배열의 자리가 삭제되거나, 길이가 줄어들지는 않는다. 이렇게 자리는 있지만 요소가 없는 부분을 배열의 홀(hole)이라고 한다.

typeof 연산자

typeof 연산자는 피연산자의 타입을 반환한다. 숫자, NaN 값은 숫자 타입, 문자열은 문자열 타입, true/false는 참/거짓(boolean) 타입, undefined는 undefined, 함수는 function, 이외는 객체 타입으로 타입을 반환한다.

instanceof 연산자

instanceof 연산자는 객체를 피연산자로 받아 특정 객체의 인스턴스인지 아닌지를 확인해준다.

피연산자가 특정 객체의 인스턴스면 true를 반환, 아니면 false를 반환한다. 인스턴스와 객체에 대한 내용은 추후 객체 파트에서 더 상세히 다뤄보자.
