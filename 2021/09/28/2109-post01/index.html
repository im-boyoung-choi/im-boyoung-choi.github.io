03. 연산자

한 달 동안 회사 일에 바빠 전혀 블로그를 신경 쓰지 못했다. JavaScript 공부해 보겠다며 이제 겨우 변수와 상수에 대해 글을 썼는데 한 달이나 공백이 생겨서 민망하기까지 하다. 그래도 잊지않고 다시 돌아온 것에 (부끄럽지만) 의미를 두고 다시 시작해보자.

오늘 다룰 주제는 연산자다. 프로그래밍 언어에서 연산자는 특정 연산을 하도록 하는 문자라고 한다. 이렇게 보면 ‘그게 뭐지’ 싶을 수도 있는데 실제 구문을 보면 누구나 ‘아 이거’ 할 것이다.

3. 논리 연산자

논리 연산자는 blooean 타입(true/false)을 위한 연산자다. 논리 연산자는 if문을 배울때 유용하게 사용된다고 하니 꼭 기억해두자.

3-1. NOT 연산자

NOT 연산자(!)는 true는 false로, false는 true로 바꿔주는 연산자이다.

3-2. AND 연산자

AND 연산자(&&)는 양 쪽의 값이 모두 true 일 때만 결과물이 true가 나온다.

3-3. OR 연산자

OR 연산자(||)는 양 쪽의 값이 하나라도 true라도 결과물이 true가 나온다. 즉, 두 값이 다 false여야 false가 나온다.

사칙연산에도 곱셈, 나눗셈 후에 덧셈, 뺄셈을 하듯이 논리 연산자에도 순서가 있다. 순서는 NOT >> AND >> OR 이다. 아래 코드를 기준으로 생각해보자.

4. 비교 연산자

비교 연산자는 두 값을 비교할 때 사용하는 연산자이다.

4-1. 두 값이 일치하는 지 확인

=== 연산자는 두 값이 일치하는 지 확인한다. 일치하면 true, 아니면 false가 나온다.

비슷한 == 연산자도 확인해보자.

== 연산자도 같은 결과값이 나왔다. 그러면 두 연산자의 차이는 무엇일까. 바로 == 연산자는 타입 검사는 하지 않는다는 것이다. 타입 검사를 하지 않으면 (1)숫자 타입의 1과 String 타입의 ‘1’, (2)0과 false, (3)null과 undefined 값을 같은 값으로 간주한다.

그래서 두 값이 일치하는 지를 확인하고 싶다면 == 연산자보다는 === 연산자 사용을 권장한다.

4-2. 두 값이 일치하지 않는지 확인

두 값이 일치하지 않는지 확인하려면 !== 연산자를 사용하면 된다.

비슷하게 != 연산자도 있지만 == 연산자와 동일하게 타입 검사를 하지 않는다.

마찬가지로 두 값이 일치하지 않는지 확인하려면 != 연산자보다는 !== 연산자 사용을 권장한다.
