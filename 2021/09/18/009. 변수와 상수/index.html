변수와 상수

특정 이름에 특정 값을 담는 것을 우리는 선언이라고 한다. 선언은 두 가지 종류로 정의할 수 있는데, 바로 변수와 상수이다.

개발에 관심이 있다면 ES6(ES2015) 이전에 사용하던 var 또는 ES6에서 추가된 let, const 중 하나는 익숙하게 느껴질 것이다. 이 중에서 어떤 것을 변수를 선언할 때 사용하고, 어떤 것을 상수를 선언할 때 사용하는지 알아보자.

1. 변수(variable)

변수(variable)란 바뀔 수 있는 값을 의미한다. 한 번 값을 선언한 뒤에도 새로운 값을 넣을 수 있다.

변수를 선언하는 키워드로 let 말고도 var 키워드가 있는데, 이제는 점점 사용하지 않는 추세이다.

var 키워드가 let 키워드와 가장 큰 차이점은 같은 이름으로 여러 번 선언할 수 있다는 것과 사용 범위 정도인데, var 키워드에 대한 정보는 아예 모르는 것이 낫다고 한다. 우리는 최대한 사용을 자제해야 하는 키워드 정도로만 우선 기억하고 넘어가자.

상수도 변수와 마찬가지로 같은 이름으로 2번을 선언하면 Identifier (상수명) has already been declared라는 오류가 발생하니 주의하자.

변수, 상수와 함께 데이터 타입에 대해 다루려고 했는데 데이터 타입은 타입 변환과 함께 다루면 내용이 너무 길어지기 때문에 다음 아티클에서 한 번에 정리하도록 하겠다.

3. 유효범위

유효 범위(scope)란 식별자(변수, 함수, 객체 등)가 접근할 수 있는 범위를 의미한다. 선언 위치에 따라 크게 지역 변수(local variavle), 전역 변수(global variable)로 구분된다.

전역 변수(global variable)

전역 변수는 함수 외부에서 선언된 변수이다. 전역 변수는 어디에서나 접근할 수 있으며, 웹 페이지가 닫혀야만 메모리에서 사라진다.

전역 변수는 함수 외부 뿐만 아니라 내부에서도 호출 가능하다.

지역 변수(local variable)

지역 변수는 함수 내에서 선언된 변수이다. 지역변수는 변수가 선언된 함수 내(하위 객체 포함)에서만 유효하며, 함수가 종료되면 메모리에서 사라진다. 함수의 매개변수 또한 함수 내에서 정의되는 지역 변수와 같이 동작한다.

함수 내에서 정의된 num 변수를 함수 밖에서 호출하면 오류가 발생한다.

이전에 사용하던 var와 달리, let const 키워드로 선언된 변수는 모든 코드 블록({})이 지역 변수가 되는 블록 레벨 스코프를 지원한다.

위에서 let과 const를 다룰 때 같은 이름으로 2번을 선언할 수는 없다고 했는데, 이 코드 블록을 기준으로 다른 블록 범위에서는 같은 이름으로 선언하는 것이 가능하다. 즉, 같은 이름의 전역 변수 선언은 오류가 나지만 같은 이름이라도 전역 변수와 지역 변수 이거나, 다른 코드 블록의 지역 변수라면 선언이 가능하다. 아래 코드를 살펴보자.
