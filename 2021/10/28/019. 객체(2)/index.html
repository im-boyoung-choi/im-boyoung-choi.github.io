객체(2)

이전 시간에는 객체의 기본 개념과 형태, 프로퍼티와 메소드에 대해 살펴봤다. 이번 시간에는 객체를 생성하는 방법과 객체 생성자 함수(constructor function)와 관련된 문법들에 대해 살펴보자.

객체를 생성하는 방법은 아래와 같다.

  1. 리터럴 표기(literal notation)를 이용한 방법
  2. 생성자 함수(constructor function)를 이용한 방법
  3. Object.create() 메소드를 이용한 방법

위 방법으로 생성되어 메모리에 대입된 객체를 인스턴스(instance)라고 한다.

1. 리터럴 표기(literal notation)를 이용한 방법

위에서 설명한 이름(key): 원하는 값(value) 형태로 프로퍼티(property)를 나열하는 방법으로, 객체를 생성하는 가장 쉬운 방법이다.

2. 생성자 함수(constructor function)를 이용한 방법

new 연산자를 이용하여 객체를 생성하고 초기화하는 방법으로, 이때 만들어지는 객체를 생성자 함수(constructor function)라고 한다.

생성자 함수와 일반 함수의 기술적 차이는 없다. 다만 생성자 함수는 아래 규칙을 따라야 한다.

  1. 함수 이름은 파스칼 표기법에 의해 첫 글자를 대문자로 시작한다.
  2. 반드시 new 연산자를 붙여 실행해야 한다.

첫 글자가 대문자로 만들어진 모든 함수에 new 연산자를 붙이면 생성자 함수가 된다. 아래 코드를 살펴보자.

new User('boyoung') 외에도 new User('철수'), new User('영희') 등을 작성하면 동일하게 사용자 객체를 만들 수 있다. 객체 리터럴을 이용하여 일일이 객체를 만드는 방법보다 훨씬 간단하고 쉽다. 재사용할 수 있는 객체 생성 코드를 구현하는 것에 생성자의 가치가 있는 것이다.

new.target

이 문법은 자주 사용되지 않는 문법이다. 나는 객체 생성자 함수에 대해 더 찾아보다가 모던 JavaScript 튜토리얼 문서를 통해 내용을 알게됐다. 간단히 살펴보고 넘어가자.

new.target 프로퍼티를 사용하면 함수가 new 연산자와 함께 호출됐는지 확인할 수 있다. 일반적인 방법으로 함수가 호출됐다면 undefined를, new 연산자로 호출했다면 함수 자체를 반환해준다. 아래 예제를 살펴보자.

이를 활용하면 일반적으로 만들어진 함수도 new 연산자를 통해 호출한 객체와 같이 동작하도록 만들 수 있다. 먼저, new 연산자 없이 생성자 함수처럼 사용하면 어떤 결과가 나오는지 알아보자.

객체 생성이 제대로 이루어지지 않아 this를 받아오지 못하고 에러가 발생한다. 그렇다면 new 연산자를 붙여 호출한 것과 같이 동작하게 만들어보자.

this
this는 메소드를 호출할 때 사용된 객체 내부에 접근할 수 있다.

JavaScript에서 this는 다른 프로그래밍 언어의 this와는 약간의 차이가 있다. JavsScript에서 this는 런타임에 결정된다. 메소드가 어디서 정의되었는지에 상관없이 점 앞에 객체가 무엇인가에 따라 자유롭게 결정된다.

this가 런타임에 결정되면 메소드를 하나만 만들어 여러 객체에서 재사용할 수 있다는 장점이 있지만, 유연함으로 인해 오류나 실수를 만들 수 있다는 단점이 있다.

화살표 함수에서는 일반 함수와 달리 고유한 this 값을 가지지 않는다. 화살표 함수에서 this를 참조하면 화살표 함수 내부가 아닌 외부 함수에서 값을 가져오기 때문에 화살표 함수에서는 this 키워드를 사용하지 않는다.

하지만 이 방법을 믿고 객체를 생성할 때 new 연산자를 생략하면 정확한 코드의 역할을 알기가 힘들다. new 연산자를 통해 호출하면 새로운 객체를 만든다는 것을 알려주는 것이기 때문에 되도록이면 정석의 방법을 사용하자.

생성자 함수의 return

생성자 함수에는 보통 return을 사용하지 않는다. 변환해야 할 것들이 this에 저장돠어 자동으로 반환되기 때문에 반환문을 명시적으로 작성할 필요가 없다. 그런데 만약 return이 사용되었다면 아래와 같은 규칙이 적용돤다.

  1. 객체를 return 한다면 this 대신 객체 반환
  2. 원시형을 return 한다면 return 무시

즉, return 뒤에 객체가 오면 해당 객체를 반환해주고 나머지 경우는 일반적인 경우와 같이 this를 반환하는 것이다. 첫 번째 규칙이 적용된 예제를 살펴보자.

return 뒤에 아무것도 명시하지 않은 경우를 살펴보자. 이 경우 두 번째 규칙이 적용된다.

3. Object.create() 메소드를 이용한 방법

Object.create() 메소드는 지정된 프로토타입(prototype) 객체와 프로퍼티(property)를 가지고 새로운 객체를 만들어준다. 따라서 이 메소드를 이용하면 사용자가 프로토타입 객체를 직접 명시할 수 있으므로 상당히 유용하게 사용된다.

Object.create() 메소드를 이용하여 객체를 생성하기 위해서는 Object.create(프로토타입 객체, [새로운 객체의 프로퍼티1, 새로운 객체의 프로퍼티1, ...]) 문법을 사용하면 된다.
