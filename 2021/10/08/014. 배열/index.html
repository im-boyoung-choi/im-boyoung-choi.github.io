배열

배열(array)은 여러 개의 항목들이 들어있는 리스트와 같다. 배열을 구성하는 각각의 값을 배열 요소(element)라고 하며, 배열 요소의 위치를 가리키는 숫자를 인덱스(index)라고 한다.

배열은 요소 타입이 고정되어 있지 않기 때문에 같은 배열에 있는 요소라도 타입이 서로 다를 수 있고, 특정 배열 요소가 비어있을 수 있어 인덱스가 연속적이지 않을 수 있으며, Array 객체로 다뤄진다는 특징을 가진다.

배열을 선언할 때는 [ ]로 값을 감싸주면 된다. 배열 안에는 어떤 값이든지 넣을 수 있다. 숫자 배열을 만들어 보자.

배열을 선언하고 나서 n 번째 항목을 조회하고 싶으면 objects[n]와 같이 조회할 수 있다. 객체 배열을 만들고, 조회해 보자.

희소 배열

희소 배열이란 배열의 속한 요소의 위치가 연속적이지 않은 배열을 말한다. 희소 배열의 경우 배열의 length 프로퍼티 값보다 배열 요소의 개수가 적다.

위 예제에서 array 배열은 9번째 자리에만 요소를 갖지만 array.length는 10이 반환되는 것을 볼 수 있다.

연관 배열

배열의 인덱스(index)로는 0을 포함한 양의 정수만을 사용한다. 이렇게 숫자로 된 인덱스 대신에 문자열로 된 키(key)를 사용하는 배열을 연관 배열이라고 한다.

대부분의 프로그래밍 언어는 연관 배열을 지원하지만 JavaScript는 별도로 연관 배열을 제공하지 않는다. 대신, 인덱스로 문자열을 사용하여 연관 배열처럼 사용할 수 있는 객체(object)를 만들 수 있다.

하지만 이렇게 생성된 배열은 Array 객체가 아닌 기본 객체로 분류되기 때문에 배열로 취급하고 다뤘을 때 정확하지 않은 값을 반환할 수 있다.

위의 예제에서 연관 배열인 array는 Array 객체가 아닌 기본 객체이기 때문에 array.length로 0을 반환한다.

ECMAScript 6부터는 이러한 불편함을 해결하기 위해 새로운 데이터 구조인 Map 객체를 제공하고 있다. 이는 다음에 자세히 살펴보자.

문자열을 배열처럼 접근하기

문자열은 변하지 않는 값이기 때문에 읽기 전용 배열로 다룰 수 있다. 즉, 배열처럼 [] 연산자를 사용하여 문자열을 구성하는 각 문자에 바로 접근할 수 있다. 또한 Array 객체가 제공하는 모든 범용 메소드와 String 객체가 제공하는 charAt() 메소드를 사용할 수 있다.

split()
string.split(separator, limit) 형태로 구현되는 split() 함수는 문자열을 ‘separator’로 잘라서, ‘limit’ 크기 이하의 배열에 저장한 뒤 반환한다. 두 매개변수는 필수 값은 아니다.

배열 여부 확인하기

배열은 별도 타입을 제공받지 않는다. typeof 연산자를 통해 배열을 확인하면 객체(object)로 반환한다.

해당 변수가 배열인지 확인하기 위해서는 아래의 방법을 사용해야 한다.

Array.isArray() 메소드

ECMAScript 5부터 Array.isArray() 메소드를 통해 배열 여부를 확인할 수 있게 됐다. 하지만 구형 브라우저에서는 지원되지 않으므로 정상적으로 동작하지 않을 수도 있다.

instanceof 연산자

Array.isArray() 메소드가 동작하지 않을 때는 instanceof 연산자를 사용해서 배열 여부를 확인할 수 있다.
