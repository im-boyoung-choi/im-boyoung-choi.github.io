전역 함수

JavaScript에서는 사용자의 편의를 위해 다양한 기능의 여러 전역 함수(predefined functions)를 미리 정의해 제공한다. 전역 함수는 어떤 타입의 객체에서도 바로 사용할 수 있다.

전역함수의 종류와 사용 방법에 대해 살펴보자.

1. eval()

eval() 함수는 문자열로 표현된 코드를 실핸하는 함수이다.

2. isNaN()

isNaN() 함수는 매개변수가 NaN 값인지 검사하여 결과를 반환한다. 매개변수가 숫자가 아니라면 숫자로 자동 변환하여 검사한다.

숫자인지 여부를 확인하는 방법은 isNaN() 함수 뿐만 아니라 typeof 연산자응 사용하는 방법도 있다.

3. isFinite()

isFinite() 함수는 매개변수가 유한한 수인지 검사하여 결과를 반환한다. 매개변수가 숫자가 아니라면 숫자로 자동 변환하여 검사한다.

4. encodeURI()와 encodeURIComponent()

encodeURI() 함수는 URI(Uniform Resource Idenrifier)에서 주소를 표시하는 특수 문자를 제외하고, 모든 문자를 이스케이프 시퀀스(escape sequences) 처리하여 부호화한다.

encodeURIComponent() 함수는 URI에서 encodeURI() 함수에서는 부호화하지 않은 모든 문자들을 전체 포함하여 이스케이프 시퀀스 처리하여 부호화한다.

URI, URL, URN

URI(Uniform Resource Identifier)는 인터넷의 우편물 주소와 같은 개념으로 정보 리소스를 고유하게 식별하고 위치를 지정할 수 있다. URI에는 URL, URN 형태가 있다.

URL(Uniform Resource Locator)은 통합 자원 지시자로 URI의 가장 흔한 형태이다. 특정 서버의 한 리소스에 대한 구체적인 위치를 서술한다. URL은 리소스가 정확히 어디에 있고 어떻게 접근할 수 있는지 분명히 알려준다.

  • https://www.google.com - google 메인 URL
  • https://www.google.com/search?q=자바스트립트 - google에 ‘자바스크립트’를 검색했을 때 URL

URN(Uniform Resource Name)은 콘텐츠를 이루는 한 리소스에 대해 그 리소스의 위치에 영향 받지 않는 유일무이한 이름 역할을 한다. 위치 독립적인 URN은 리소스를 여기저기 옮기더라도 문제없이 동작한다.

  • urn:irft:rfc:2141 - ‘RFC 2141’ 문서

이스케이프 시퀀스(escape sequences)
백슬래시( \ ) 뒤에 한 문자나 숫자 조합이 오는 문자 조합을 의미한다. 이스케이프 시퀀스는 단일 문자로 간주되므로 문자 상수로 유효하다.

4. decodeURI()와 decodeURIComponent()

decodeURI() 함수는 encodeURI() 함수로 부호화된 URI를 해독한다.

decodeURIComponent() 함수는 encodeURIComponent() 함수로 부호화된 URI를 해독한다.

5. escape()와 unescape()

escape() 함수는 전달받은 문자열에서 특정 문자들을 16진법 이스케이프 시퀀스 문자로 변환한다.

unescape() 함수는 escape() 함수로 변환된 16진법 이스케이프 시퀀스 문자를 원래 형태로 변환한다.
