배열 안에 있는 값이 숫자, 문자열, 또는 boolean 값일 때, 몇 번째 원소인지 찾기 위해서는 indexOf 함수를 사용하면 된다. 하지만 배열 안에 있는 값이 객체나, 배열이라면 indexOf는 사용할 수 없다.

5. find

find 함수는 위에서 배운 findIndex 함수와 비슷하다. 단, 찾아낸 값이 몇 번째인지 알아내는 것이 아니라, 찾아낸 값 자체를 반환한다.

filter 함수 인자로는 조건을 검사하는 함수를 넣어주며, 각 원소의 값을 받아오게 된다. 우리가 방금 작성한 코드는 앞서 배운 논리 연산자를 활용해 아래와 같이 표현할 수도 있다.

9. shift와 pop

shift와 pop 함수는 비슷하지만 다르다. shift 함수는 첫 번째 원소를 배열에서 추출해 준다. 이때, 추출하는 과정에서 배열 내에 해당 원소가 삭제된다.

pop 함수는 마지막 원소를 배열에서 추출해 준다. 마찬가지로 추출하는 과정에서 배열 내에 해당 원소가 삭제된다.

13. reduce

reduce 함수는 잘 사용하면 아주 유용한 내장함수이지만, 오늘 살펴본 내장함수들 중에 가장 까다롭고 복잡하다. 나도 벨로퍼트 님의 강의 문서만 따라가서는 이해가 안돼서 여러 사이트를 추가로 참고했다. 그 내용을 함께 정리해두었으니 바로 이해가 되지 않더라도 끝까지 따라가보길 바란다.

reduce 함수는 배열.reduce((누적값, 현재값, 인덱스, 원 배열) => { return 결과 }, 초기값) 형태로 사용된다. 여기서 첫 번째 요소가 ‘이전값’이 아닌 ‘누적값’이라는 것에 주의하자. 3 번째 매개변수인 ‘인덱스’는 현재 배열 내 처리돠고 있는 요소의 인덱스, 4 번째 매개변수인 ‘원 배열’은 reduce 함수에서 호출되어 사용하고 있는 원 배열을 의미한다.

reduce 함수는 반복 작업이 필요한 대부분의 경우에 사용할 수 있다고 생각하면 된다. reduce 함수를 사용해서 평균을 계산할 수도 있다. 평균을 계산하기 위해서는 전체 숫자를 더하고, 배열의 length 를 확인해 나눠주면 된다. 다음 코드를 살펴보자.

  2. 두 번째 {acc: 3, cur: 3, idx: 2, arr: Array(5)}가 각 요소의 값으로 들어온다. if 문 조건이 성립하지 않기 때문에 acc + cur을 실행, 다음 acc 값으로 6 을 반환한다.

  3. 세 번째 {acc: 6, cur: 4, idx: 3, arr: Array(5)}가 각 요소의 값으로 들어온다. if 문 조건이 성립하지 않기 때문에 acc + cur을 실행, 다음 acc 값으로 10 을 반환한다.
