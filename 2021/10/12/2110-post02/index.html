06. 객체

JavaScript를 공부하면서 뭔가 막히거나 내가 제대로 이해한 게 맞나 싶을 때는 항상 객체를 공부하고 있는 순간이었다. 따라가고는 있는데 이걸 내가 실무에서 사용할 수 있을지 의문이 남는다.

그렇기 때문에 특히 객체는 무작정 따라 해본다는 생각보다는 틈틈이 제대로 이해하고 있는지 확인하면서 공부하는 걸 추천한다.

객체는 우리가 변수 혹은 상수를 사용할 때 하나의 이름에 여러 종류의 값을 넣을 수 있게 해준다. 아래 코드를 살펴보자.

4. Getter 함수와 Setter 함수

객체 안에 Getter 함수와 Setter 함수를 설정하는 방법을 알아보자. 객체를 만들고 나면 다음과 같이 객체 안의 값을 수정할 수도 있다.

Getter 함수와 Setter 함수를 사용하게 되면 특정 값을 바꾸려고 하거나, 특정 값을 조회하려고 할 때 우리가 원하는 코드를 실행시킬 수 있다. 아래 코드를 살펴보자.

우리는 numbers.sum()을 실행한 게 아니라, numbers.sum을 조회했을 뿐인데 함수가 실행되고 그 결과값이 출력됐다. 이런 식으로 Getter 함수는 특정 값을 조회할 때 우리가 설정한 함수로 연산된 값을 반환한다.

이번에는 Setter 함수에 대해 살펴보자.

Setter 함수를 설정하고 numbers.a = 5라고 값을 설정했을 때 5를 함수의 파라미터로 받아오게 된다. 위 코드에서는 객체 안에 _a, _b를 선언해 주고, 이 값들을 위한 Getter 함수와 Setter 함수를 설정해 줬다.

이전에 만든 객체에서는 numbers.sum이 조회될 때마다 덧셈이 이루어진 반면, 이번에는 a 혹은 b 값이 바뀔 때마다 sum 값을 연산하는 것을 볼 수 있다.
