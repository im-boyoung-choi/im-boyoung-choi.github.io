함수

함수(function)는 특정 코드를 하나의 명령으로 실행할 수 있게 한다. 함수는 필요할 때마다 호출해 해당 작업을 반복해서 수행할 수 있다.

JavaScript에서는 함수도 하나의 데이터 타입이다. 따라서 함수를 변수에 대입하거나, 함수에 프로퍼티를 지정하는 것도 가능하다. 또한 함수는 다른 함수 내에 중첩해서 사용할 수도 있다.

함수 이름은 함수를 구분하는 식별자(idenrifier)로 사용되며, 매개변수(parameter)는 함수를 호출할 때 인수(argument)로 전달된 값 받아 함수 내부에서 사용할 수 있게 해주는 변수이다.

1. 매개변수(parameter)

매개변수(parameter)는 함수를 호출할 때 인수(argument)로 전달된 값을 받아 함수 내부에서 사용할 수 있게 해주는 변수이다. 함수를 정의할 때는 매개변수 타입을 따로 명시하지 않으며, 호출할 때도 인수(argumaent)에 대해 타입 검사를 진행하지 않는다.

함수를 호출할 때 받아야 하는 매개변수의 수보다 적은 수의 인수를 전달하더라도 오류가 발생하지 않으며, 값을 전달받지 못한 매개변수는 자동으로 undefined 값으로 설정된다.

아래 예제는 3개의 매개변수를 가지는 함수에 각각 다른 수의 인수를 전달하는 예제이다.

위 예제에서 sum() 함수를 호출할 때, 인수가 3개 미만으로 전달되면 나머지 항목에는 undefined 값이 설정되기 때문에 산술 연산을 수행할 수 없어서 NaN 값을 반환한다. 만약 전달된 인수만을 가지고 정상적으로 계산하고 싶다면 아래와 같이 함수를 변경해야 한다.

매개변수의 수보다 많은 수의 인수가 전달되면 매개변수에 대입되지 못하고 남는 인수들은 참조할 방법이 없어진다.

이때 arguments 객체를 이용하면 함수가 호출될 때 전달되는 인수를 배열의 형태로 저장할 수 있다. 또한 인수의 총 개수를 arguments 객체의 length로 확인할 수도 있다.

아래 예제의 sum() 함수는 전달받은 인수의 개수에 상관없이 언제나 정상적인 계산을 수행한다.

arguments 객체
함수를 호출할 때 인수들과 함께 암묵적으로 함수에 전달되는 객체이다. 실제 배열이 아닌 마치 배열 형태처럼 숫자로 인덱싱된 프로퍼티를 가진 유사 배열 객체이다. 즉, 숫자로 된 인덱스와 length 프로퍼티를 가지지만 모든 것을 배열처럼 다룰 수는 없다.

디폴트 매개변수(default parameter)
디폴트 매개변수는 함수를 호출할 때 명시된 인수를 전달하지 않았을 경우 사용하게 될 기본값을 의미한다. 원래의 기본값은 undefined이지만, 디폴트 매개변수를 이용하면 기본값을 바꿀 수 있다. (크롬, 파이어폭스 브라우저 지원)

나머지 매개변수(rest parameter)
생략 접두사(…)를 사용해 특정 위치의 인수부터 마지막 인수까지 한 번에 배열에 저장하는 것을 의미한다. (엣지, 크롬, 사파리, 파이어폭스, 오페라 브라우저 지원)

4. 화살표 함수

함수를 선언하는 방식 중 또 다른 방법은 화살표 함수 문법을 사용하는 것이다.

function 키워드 대신에 => 문자를 사용해서 함수를 구현하는 방법이다. 화살표의 좌측에는 매개변수, 화살표의 우측에는 코드 블록이 들어온다.
