함수

함수(function)는 특정 코드를 하나의 명령으로 실행할 수 있게 한다. 함수는 필요할 때마다 호출해 해당 작업을 반복해서 수행할 수 있다.

JavaScript에서는 함수도 하나의 데이터 타입이다. 따라서 함수를 변수에 대입하거나, 함수에 프로퍼티를 지정하는 것도 가능하다. 또한 함수는 다른 함수 내에 중첩해서 사용할 수도 있다.

배열

배열(array)은 여러 개의 항목들이 들어있는 리스트와 같다. 배열을 구성하는 각각의 값을 배열 요소(element)라고 하며, 배열 요소의 위치를 가리키는 숫자를 인덱스(index)라고 한다.

배열은 요소 타입이 고정되어 있지 않기 때문에 같은 배열에 있는 요소라도 타입이 서로 다를 수 있고, 특정 배열 요소가 비어있을 수 있어 인덱스가 연속적이지 않을 수 있으며, Array 객체로 다뤄진다는 특징을 가진다.

반복문

반복문(iteration sstatements)이란 똑같은 명령을 일정 횟수만큼 반복하여 수행하도록 제어하는 실행문이다.

프로그램이 처리하는 대부분의 코드는 반복적인 형태가 많기 때문에 가장 많이 사용되는 실행문 중 하나다. 반복문의 종류와 쓰임에 대해 알아보자.

조건문

프로그램의 순차적인 흐름을 제어해야 할 때 사용하는 실행문을 제어문(control flow statements)이라고 한다. 이러한 제어문에는 조건문, 반복문 등이 포함된다.

그 중 이번 시간에는 조건문에 대해 살펴볼 예정이다. 조건문(conditional statements)이란 주어진 표현식의 결과에 따라 별도의 명령을 수행하도록 제어하는 실행문이다. 조건문의 종류와 쓰임에 대해 알아보자.

hello, JavaScript!

처음 블로그를 만든 목적은 패스트캠퍼스 ‘프론트엔드 개발 올인원 패키지’ 강의 중 JavaScript 파트인 ‘벨로퍼트와 함께하는 모던 자바스크립트’를 수강하며 개발 공부를 하는 것을 기록하기 위함이었다.

벨로퍼트 님의 GitBook을 참고해서 ‘1장. 자바스크립트 입문’의 마지막 파트를 공부하던 중 객체를 너무 겉핥기로 공부했다는 것을 알게됐다. 추가로 객체에 대해 공부하다 보니 벨로퍼트 님의 강의 문서만 따라해서는 초보자가 JavsScript를 쉽게 이해하기는 어려울 수 있다는 생각이 들었고, 작성했던 11개의 게시글을 지우고 다시 처음부터 정리를 시작한다.

벨로퍼트 님의 강의 문서를 토대로 작성하려고 했던 처음 목표와는 달리 나는 좀 더 초보자를 위해 TCP School 문서를 토대로 개념을 정리해가려 한다. 벨로퍼트 님의 강의에서 다루는 내용이 빠지는 것은 아니니 2개의 문서를 함께 참고하길 바란다.
